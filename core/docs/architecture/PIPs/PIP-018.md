# **PIP-018: Edge-Vision PII Obfuscation Standard**

| **PIP Number** | **018** |

| **Title** | Edge-Vision PII Obfuscation Standard |

| **Author** | PAN Command / Network Engineering |

| **Status** | Active (Mesa Pilot) |

| **Type** | Standards Track |

| **Created** | 2026-02-12 |

## **1\. Abstract**

To strictly enforce the Proxy Agent Network's (PAN) Zero-Biometric Privacy Policy and comply with Maricopa County privacy expectations, this proposal mandates a local, edge-vision middleware layer. This layer automatically detects and redacts incidental **Personally Identifiable Information (PII)** from any environmental photos captured by Vanguard Agents prior to encryption and transmission to Fleet Operators.

## **2\. Motivation**

Vanguard Agents execute the Optical Reclamation Protocol (ORP) in live, public environments (e.g., busy intersections, parking lots). While Fleet Operators occasionally require high-resolution photos of permanently damaged sensor arrays (e.g., a shattered LiDAR dome) for insurance claims, these photos inherently run the risk of capturing civilian bystanders, other drivers, or private property.

Transmitting unredacted civilian PII to Fleet Operators creates an unacceptable legal liability. Therefore, redaction must happen at the physical edge—on the Agent's mobile device—*before* the image is encrypted via PIP-016 standards.

## **3\. Technical Specifications**

### **3.1 Edge-Vision Pipeline**

Nodes (the PAN Tactical App running on iOS/Android) must process all captured visual artifacts locally. **Cloud-based image processing or API calls for PII detection are strictly forbidden.**

* **iOS Architecture:** Apple Vision Framework / CoreML.  
* **Android Architecture:** Google ML Kit (On-Device Vision API).

### **3.2 Mandatory Redaction Targets**

The edge model must identify and obfuscate the following elements with a heavy Gaussian blur (minimum 30px radius) or a solid black \#000000 bounding box:

1. **Human Faces:** All pedestrian and driver faces within the frame.  
2. **License Plates:** All vehicle license plates (the model may exempt the specific target\_vin or Fleet ID of the stranded Autonomous Vehicle being serviced).  
3. **Residential Markers:** Visible residential address numbers or private signage in the background.

### **3.3 The Execution Workflow**

1. Vanguard Agent captures a photo of the damaged AV sensor.  
2. The image is held in volatile RAM.  
3. The Edge-Vision Middleware sweeps the image and applies irreversible blurring to all detected PII bounding boxes.  
4. The redacted image is encrypted using the Fleet Operator's RSA-OAEP Public Key (Per PIP-016).  
5. The encrypted payload is transmitted to the PAN Gateway.  
6. The unredacted source image is permanently zero-filled from the mobile device's memory.

## **4\. Compliance & Slashing**

The PAN Gateway performs randomized heuristic checks on encrypted payload sizes to detect bypass attempts. Furthermore, Fleet Operators are legally bound by their Master Service Agreement (MSA) to report any unredacted PII that slips through the network.

If a Vanguard Agent is found to be utilizing tampered hardware or modified app binaries to bypass the Edge-Vision middleware, it constitutes a **Tier-2 Privacy Violation**. The Agent will suffer an immediate \-25 VTS (Vanguard Trust Score) penalty, and the specific Fleet Operator may permanently ban the Agent from their specific dispatch routing pool.
