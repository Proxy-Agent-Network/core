# **PIP-016: RSA-OAEP Telemetry Encryption Standard**

| **PIP Number** | **016** |

| **Title** | RSA-OAEP Telemetry Encryption Standard |

| **Author** | PAN Command / Network Engineering |

| **Status** | Active (Mesa Pilot) |

| **Type** | Standards Track |

| **Created** | 2026-02-12 |

## **1\. Abstract**

This proposal mandates the use of **RSA-OAEP (Optimal Asymmetric Encryption Padding)** for all proprietary Fleet Operator data captured during an intervention. This ensures that only the Fleet Operator (e.g., Waymo, Zoox) can access raw execution telemetry and hardware condition photos, maintaining a strict Zero-Knowledge architecture for the PAN Gateway and protecting Fleet trade secrets.

## **2\. Motivation**

Autonomous Vehicle (AV) fleets consider their hardware failure rates, proprietary Unified Diagnostic Service (UDS) codes, and high-fidelity sensor imagery to be highly classified trade secrets. If the Proxy Agent Network (PAN) stores this intervention data in plaintext, it creates a massive honeypot and a critical liability risk. By enforcing edge-encryption on the Vanguard Agent's device, PAN ensures it acts solely as a blind, secure routing layer.

## **3\. Technical Specifications**

### **3.1 Cryptographic Parameters**

* **Encryption Scheme:** Hybrid Encryption (RSA-OAEP for key wrapping \+ AES-256-GCM for payload).  
* **RSA Key Size:** 4096-bit minimum.  
* **Hash Function:** SHA-256.

### **3.2 The Edge-Encryption Workflow**

1. **Key Distribution:** The Fleet Operator includes their RSA Public Key in the initial /v2026.1/fleet/dispatch webhook payload.  
2. **Artifact Capture:** The Vanguard Agent executes the Optical Reclamation Protocol (ORP) and captures necessary compliance artifacts (e.g., UWB distance, TPM signatures, and optional exterior sensor condition photos).  
3. **Edge Encryption:** The PAN Tactical App encrypts the proprietary artifacts locally in the device's volatile memory (RAM) using the Fleet Operator's Public Key.  
4. **Transmission & Purge:** The encrypted payload is uploaded to the PAN Gateway. The Tactical App immediately and permanently zero-fills the unencrypted data from the mobile device.

## **4\. SB 1417 Public vs. Private Payloads**

To comply with both privacy standards and Arizona State Law, the payload is bifurcated:

* **The Public OHR (Unencrypted):** The core Optical Health Report required by Arizona SB 1417 (Agent ID, Timestamp, UWB Lock boolean, TPM Signature) remains readable by PAN for compliance auditing and L402 settlement verification.  
* **The Proprietary Telemetry (Encrypted):** Exact UDS diagnostic outputs, high-resolution hardware photos, and exact Fleet internal vehicle identifiers are encrypted and mathematically inaccessible to PAN Command.

## **5\. Security & Liability Considerations**

By utilizing RSA-OAEP at the edge, PAN protects against Man-in-the-Middle (MITM) attacks and ensures that Vanguard Agents cannot leak proprietary fleet sensor data (e.g., a rogue Agent cannot sell photos of a damaged, unreleased AV sensor array to the press, as the photos are encrypted the millisecond they are captured). This firmly establishes PAN's liability shield.
