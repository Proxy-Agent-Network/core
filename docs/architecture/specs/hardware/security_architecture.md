**Status:** Active (Mesa Pilot)

**Target:** Fleet Security Teams, CISOs, & Core Protocol Engineers

## **Abstract**

To prevent "Phantom Agent" fraud (e.g., GPS spoofing, emulator farms, or account sharing), the Proxy Agent Network (PAN) strictly prohibits software-only authentication. PAN relies on a **Hardware Root of Trust**. This document details how Vanguard Agent mobile devices (utilizing Apple Secure Enclave or Android TPM 2.0 / StrongBox) cryptographically sign physical proofs, interact with the L402 settlement engine, and ensure private keys never enter vulnerable system RAM.

## **1\. The Silicon Identity (Key Generation Ceremony)**

Unlike standard ride-share or gig-economy apps, a Vanguard Agent's identity key is never generated by the mobile OS software, nor is it stored in a cloud database.

1. **On-Boarding (Mesa Hub):** During the physical provisioning sequence at the Mesa Hub, the PAN Tactical App sends an instruction to the mobile device's secure coprocessor.  
2. **Hardware Generation:** The Apple Secure Enclave or Android StrongBox generates an ECDSA secp256r1 keypair entirely inside its shielded silicon.  
3. **Public Key Export:** The Public Key is exported and registered in the PAN Identity Registry, permanently mapped to the Vanguard Agent's vetted profile (DD-214/Background Check).  
4. **Hardware Sealing:** The Private Key is mathematically fused to the silicon. It is strictly non-exportable. It cannot be extracted via USB debugging, cloned to another device, or backed up to iCloud/Google Drive.

## **2\. L402 Mission Signing & Execution**

The Vanguard Mobile Node operates as a localized hardware signer for the L402 Lightning settlement layer.

* **The Challenge:** To settle a mission and claim the Satoshi bounty, the Agent must prove physical presence (UWB lock) and execution intent.  
* **The Cryptographic Flow:**  
  1. The Autonomous Vehicle (AV) locks the Satoshi bounty in an L402 HODL Invoice routed to the PAN Gateway.  
  2. The Agent arrives on-site and establishes an Ultra-Wideband (UWB) Time-of-Flight proximity lock (\< 1.5 meters).  
  3. The PAN Tactical App constructs a JSON payload containing the mission\_id, UWB distance, and a high-precision UTC timestamp.  
  4. The app passes the SHA-256 hash of this payload into the Secure Enclave.  
  5. The Secure Enclave signs the hash with the non-exportable private key.  
  6. The signed payload is broadcast to the PAN Gateway. Once verified, and combined with the AV's FAULT\_CLEARED webhook, the L402 Preimage is released, instantly settling the funds to the Agent's wallet.

## **3\. Biometric Gating (The Human-Hardware Bond)**

A stolen PAN-issued device is useless to a malicious actor. The hardware private key cannot automatically sign payloads.

* **Zero-Knowledge Biometrics:** Every critical action (e.g., swiping ACCEPT\_MISSION or ORP\_COMPLETE) requires active biometric gating.  
* **Execution:** The Agent must authenticate via FaceID or biometric fingerprint. The Secure Enclave evaluates the biometrics locally. If successful, the Enclave unlocks the private key for a single transaction signature.  
* *Note: PAN never ingests, stores, or processes the raw biometric data. We only receive the resulting cryptographic signature proving local biometric success.*

## **4\. Continuous Kernel Attestation (Anti-Tamper)**

Because Vanguard Agents operate in untrusted physical environments, the PAN network must continuously verify that the mobile OS kernel has not been compromised to feed fake GPS or UWB data to the app.

### **The Virtual "Dead Man's Switch"**

During an ACTIVE\_PATROL shift, the PAN App silently executes continuous integrity checks:

1. **Polling:** The app polls Apple DeviceCheck and Android Play Integrity API every 5 minutes (or 60 seconds if the device is on a 7-day provisioning probation).  
2. **Validation:** These native OS APIs verify the bootloader state, kernel integrity, and app binary signatures.  
3. **Automatic Slashing:** If the APIs detect a rooted kernel, a jailbroken OS, an unlocked bootloader, or an emulated environment (e.g., Bluestacks):  
   * The PAN Tactical App instantly wipes its local cache.  
   * The PAN Gateway immediately invalidates the device's Public Key in the Identity Registry.  
   * The Node is effectively "bricked" from the network and the Agent is permanently revoked.
